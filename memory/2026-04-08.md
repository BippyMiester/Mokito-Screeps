# Session: 2026-04-08 - Mokito Screeps Bot Development

## Summary
Major development session focused on completing multi-room harvesting, defense structures, server management, and various bug fixes for the Mokito Screeps bot.

## Completed Tasks

### Multi-Room Harvesting Implementation
- Created **RemoteHarvester** role (`src/roles/RemoteHarvester.js`)
  - Mines energy from sources in adjacent rooms
  - Builds containers near sources to store energy
  - Travels to target room and operates autonomously

- Created **Hauler** role (`src/roles/Hauler.js`)
  - Transports energy from remote room containers to home room
  - Delivers to storage, extensions, spawn, or containers
  - Returns to remote room when empty

- Created **Claimer** role (`src/roles/Claimer.js`)
  - Reserves controller in remote rooms to prevent source decay
  - Returns home when TTL is low to be recycled

- Updated **RoomManager** to handle remote room management
  - Auto-scouts adjacent rooms every 100 ticks
  - Manages remote assignments and worker counts
  - Spawns remote workers based on needs

- Updated **SpawnManager** with remote worker spawning priorities
  - Added spawnRemoteHarvester(), spawnHauler(), spawnClaimer() methods
  - New body cost calculations for remote roles

### Creep Ratio Updates
- Changed Builders:Upgraders to **1:1 ratio**
- Changed Repairers to **(Builders + Upgraders) / 2**
- Updated both spawning logic and priority display

### Defense Structures
- Added **tower** construction (RCL 3+)
- Added **rampart** construction around spawn/controller (RCL 4+)
- Added **wall** construction at room exits (RCL 5+)
- Updated Builders to prioritize defense construction
- Updated Repairers to prioritize defense structures

### Obstacle Avoidance for Harvesters
- Implemented move request system for non-harvester creeps
- Harvesters detect blocking creeps and request them to move
- After 3 ticks stuck, forcefully push blocking creeps
- Move requests expire after 5 ticks
- All creeps (Runner, Builder, Repairer, Upgrader) respond to move requests

### Idle Upgrade Behavior
- All creeps now upgrade controller when primary task unavailable
- Builders upgrade when no construction sites or repairs needed
- Repairers upgrade when no structures need repair
- Runners upgrade when has energy but nothing to deliver
- Ensures maximum efficiency, no idle time

### Wait for Full Energy Before Spawning
- SpawnManager now waits for full energy capacity before spawning (for maximum body parts)
- Status shown in multi-line heartbeat console output
- Emergency spawning bypasses this check when harvesters < 2

### Heartbeat Improvements
- Multi-line console output with clear separators
- Shows remote worker counts (RH, Ha, C) only when > 0
- Shows energy waiting status only when relevant
- Removed spammy "Waiting for full energy" messages

## User Preferences Noted
- Only use CLI reset command for future resets (no manual db.json manipulation)
- Don't restart server unless explicitly told
- Multi-line console output preferred for readability
- Keep Steam API key in .env file
- All creeps should upgrade when idle

## Challenges Encountered & Solved
1. **Reset Script Issue** - Original CLI reset script timed out. Solved by creating bash script alternative.
2. **Empty Database Issues** - Creating empty JSON caused server errors. Solved by using `npx screeps init`.
3. **Server Not Starting** - Needed Steam API key in systemd service. Fixed by adding to .env and service file.
4. **Database Still Had Data** - Screeps regenerates db.json on startup. Must be fully stopped before wiping.

## Learned Lessons
- Screeps server creates terrain data automatically, don't try to fully empty database
- Systemd service needs proper environment variables set
- CLI timeout is normal, just shows connection closed
- Tick rate setting returns 'OK' when successful
